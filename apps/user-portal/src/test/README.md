# Руководство по тестированию User Portal

## Обзор

Проект использует двойной подход к тестированию:
- **Unit тесты**: Проверяют конкретные примеры и edge cases
- **Property-based тесты**: Проверяют универсальные свойства корректности

## Property-Based тестирование

Property-based тесты используют библиотеку `fast-check` для генерации множества тестовых случаев и проверки универсальных свойств системы.

### Конфигурация

- **Минимум итераций**: 100 для каждого property теста
- **Таймаут**: 5 секунд на тест
- **Генераторы**: Кастомные генераторы для доменных объектов

### Структура тегов

Все property тесты должны использовать следующий формат тегов:
```
Feature: user-portal, Property N: Описание свойства
```

Где N - номер свойства из документа проектирования.

### Примеры property тестов

#### Валидация (validation.property.test.ts)
- Property 2: Email валидация
- Property 3: Валидация минимальной длины пароля  
- Property 4: Валидация совпадения паролей

#### Аутентификация (auth.property.test.ts)
- Property 10: Сохранение токенов в localStorage при "Запомнить меня"
- Property 11: Сохранение токенов в sessionStorage без "Запомнить меня"
- Property 12: Автоматическое добавление токена в заголовки

#### Адаптивность (responsive.property.test.ts)
- Property 22: Адаптивность для мобильных устройств
- Property 23: Адаптивность для планшетов
- Property 24: Адаптивность для десктопов
- Property 25: Динамическая адаптация при изменении размера

### Запуск тестов

```bash
# Запуск всех тестов
npm test

# Запуск в режиме наблюдения
npm run test:watch

# Запуск с UI интерфейсом
npm run test:ui
```

### Генераторы данных

Файл `fast-check.config.ts` содержит:
- Кастомные генераторы для доменных объектов
- Утилитарные функции для проверки свойств
- Глобальные настройки для всех property тестов

### Принципы написания property тестов

1. **Универсальность**: Свойство должно выполняться для всех допустимых входных данных
2. **Детерминированность**: Результат должен быть предсказуемым
3. **Независимость**: Тесты не должны зависеть друг от друга
4. **Читаемость**: Свойство должно быть понятно из названия теста

### Отладка property тестов

При падении property теста fast-check предоставляет:
- Минимальный контрпример, вызвавший ошибку
- Путь сжатия к этому примеру
- Возможность воспроизвести конкретный случай

Для воспроизведения конкретного случая используйте:
```typescript
fc.assert(
  fc.property(generator, (input) => {
    // тест
  }),
  { seed: 123456789 } // seed из вывода ошибки
)
```

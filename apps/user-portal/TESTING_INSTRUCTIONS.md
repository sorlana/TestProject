# Инструкция по тестированию Google авторизации

## Быстрый старт

1. **Запустите приложение:**
   ```bash
   cd apps/user-portal
   npm run dev
   ```

2. **Откройте браузер:**
   - Перейдите на http://localhost:5173
   - Откройте Developer Tools (F12)
   - Перейдите на вкладку Console

3. **Проверьте логи:**
   В консоли должны появиться сообщения:
   ```
   🔧 Переменные окружения:
   VITE_GOOGLE_CLIENT_ID: 986072857181-k77rcjis4dat7f3u60r2t120co6fsse3.apps.googleusercontent.com
   VITE_API_BASE_URL: http://localhost:8081
   🔄 Начинаем загрузку Google Identity Services...
   📥 Загружаем скрипт Google Identity Services...
   ✅ Google Identity Services успешно загружен
   🔧 Инициализируем Google Identity Services...
   ✅ Google Identity Services успешно инициализирован
   ```

4. **Тестируйте кнопку:**
   - Перейдите на страницу логина (/app/login)
   - Нажмите кнопку "Войти через Google"
   - В консоли должно появиться: `🖱️ Клик по кнопке Google авторизации`
   - Должно открыться окно Google авторизации

## Возможные проблемы

### Проблема: Переменные окружения undefined
**Решение:** Убедитесь, что файл `.env` существует и содержит правильные значения.

### Проблема: Google Identity Services не загружается
**Решение:** 
- Проверьте интернет соединение
- Отключите блокировщики рекламы
- Попробуйте другой браузер

### Проблема: "Invalid client ID"
**Решение:** Проверьте правильность Client ID в Google Cloud Console.

### Проблема: Кнопка неактивна
**Решение:** Дождитесь полной загрузки Google Identity Services (смотрите логи в консоли).

## Дополнительные тесты

### Тест 1: Автономный HTML файл
Откройте `test-google-auth.html` в браузере для проверки базовой функциональности.

### Тест 2: Проверка API
Убедитесь, что Auth Service запущен на порту 8081:
```bash
curl http://localhost:8081/health
```

### Тест 3: Проверка CORS
Убедитесь, что CORS настроен правильно для домена localhost:5173.

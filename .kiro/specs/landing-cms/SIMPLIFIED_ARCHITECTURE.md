# Упрощенная архитектура лендинга

## Ключевые изменения

Лендинг на Piranha CMS теперь является **простым контентным сайтом** без логики аутентификации.

## Что ЕСТЬ в лендинге

1. **Piranha CMS** для управления контентом
2. **Razor Pages** для рендеринга страниц
3. **Простые редиректы** на фронтенд-микросервис
4. **Статический контент**: Hero, О программе, Тарифы
5. **Piranha Manager** для редактирования контента

## Чего НЕТ в лендинге

1. ❌ React компонентов
2. ❌ Форм авторизации/регистрации
3. ❌ Интеграции с Auth Service
4. ❌ JWT токенов и cookies
5. ❌ Логики аутентификации
6. ❌ AuthController, AuthService, AuthenticationMiddleware

## Компоненты лендинга

### 1. LandingController

```csharp
public class LandingController : Controller
{
    private readonly IApi _api;
    
    public async Task<IActionResult> Index()
    {
        var page = await _api.Pages.GetBySlugAsync<LandingPage>("/");
        return View(page);
    }
}
```

### 2. TariffController

```csharp
public class TariffController : Controller
{
    // Простой редирект на фронтенд-микросервис
    public IActionResult Purchase(string tariffId)
    {
        return Redirect($"/app/payment?tariffId={tariffId}");
    }
}
```

### 3. Razor Views

- `Index.cshtml` - главная страница с Hero, О программе, Тарифами
- `_Layout.cshtml` - навигация с кнопкой "Личный кабинет" → `/app`

### 4. Piranha CMS Models

- `LandingPage` - модель главной страницы
- `HeroRegion` - Hero секция
- `AboutRegion` - секция "О программе"
- `TariffsRegion` - секция "Тарифы"
- `TariffItem` - элемент тарифа

## Маршрутизация

```
/                    → Landing CMS (главная страница)
/manager             → Piranha Manager (админка)
/tariff/purchase/:id → Редирект на /app/payment?tariffId=:id

/app                 → User Portal (фронтенд-микросервис)
/app/login           → User Portal (авторизация)
/app/register        → User Portal (регистрация)
/app/dashboard       → User Portal (личный кабинет)
/app/payment         → User Portal (оплата)
```

## Взаимодействие с фронтенд-микросервисом

Лендинг просто делает редиректы:

1. **Кнопка "Личный кабинет"** → `<a href="/app">Личный кабинет</a>`
2. **Кнопка "Купить"** → `<a href="/app/payment?tariffId={id}">Купить</a>`

Фронтенд-микросервис (/app) сам решает:
- Показать страницу авторизации, если пользователь не авторизован
- Показать страницу оплаты, если пользователь авторизован

## Преимущества упрощенной архитектуры

1. **Разделение ответственности**: Лендинг = контент, Фронтенд = логика
2. **Простота**: Нет сложной интеграции с Auth Service
3. **Независимость**: Лендинг и фронтенд развиваются независимо
4. **Масштабируемость**: Каждый микросервис масштабируется отдельно
5. **Безопасность**: Лендинг не хранит токены и пароли

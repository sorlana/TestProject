# План реализации: Микросервис аутентификации пользователей

## Обзор

Данный план описывает пошаговую реализацию микросервиса аутентификации пользователей на ASP.NET Core 8.0 с использованием чистой архитектуры. Микросервис обеспечивает регистрацию, аутентификацию через логин/пароль и Google OAuth, управление профилями, подтверждение телефона через SMS и интеграцию с внешним nopCommerce API для управления подписками.

Микросервис располагается в структуре проекта по адресу: `apps/user-authentication-service/`

**Внешние зависимости:**
- PostgreSQL (база данных)
- Redis (кеширование и rate limiting)
- RabbitMQ (обмен сообщениями между микросервисами)
- Google OAuth API (аутентификация через Google)
- SMS провайдер API (подтверждение телефона)
- nopCommerce API (внешний сервис управления подписками, развернут отдельно)

## Примечания

- Микросервис располагается в `apps/user-authentication-service/` согласно структуре проекта
- Задачи, помеченные `*`, являются опциональными и могут быть пропущены для более быстрого MVP
- Каждая задача ссылается на конкретные требования для отслеживаемости
- Checkpoint задачи обеспечивают инкрементальную валидацию
- Property-based тесты не включены, так как в design.md отсутствует раздел Correctness Properties
- Unit тесты и integration тесты помечены как опциональные для ускорения разработки
- Все тесты должны быть реализованы перед продакшен развертыванием
- nopCommerce развернут как внешний сервис, интеграция через HTTP API с retry политикой
- Используется существующая инфраструктура проекта (PostgreSQL, Redis, RabbitMQ из docker-compose.yaml)
